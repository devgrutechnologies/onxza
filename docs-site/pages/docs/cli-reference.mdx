# CLI Reference

> Complete command reference for the ONXZA CLI v0.1.0.

## Global Options

| Flag | Description |
|---|---|
| `-v, --version` | Print ONXZA CLI version |
| `--json` | Output in JSON format (machine-readable) |

## `onxza init`

Initialize a new ONXZA installation — creates workspace directories, `openclaw.json`, governance script stubs, a pre-commit TORI hook, and a safety checkpoint.

```bash
onxza init [--dir <path>] [--company <name>] [--no-git] [--json]
```

Idempotent — safe to re-run on an existing installation.

## `onxza status`

Show ONXZA system status: registered agents, ticket counts, health score.

```bash
onxza status [--agent <id>] [--company <name>] [--no-color]
```

| Option | Description |
|---|---|
| `--agent <id>` | Show details for a specific agent |
| `--company <name>` | Filter agent view by company |
| `--no-color` | Disable color output |

## `onxza agent`

Manage ONXZA agents.

| Command | Description |
|---|---|
| `onxza agent create <Company_Dept_Role>` | Scaffold a new agent workspace (validates naming, writes 6 files, TORI-QMD gate, registers, checkpoints) |
| `onxza agent list` | List all registered agents with current status |
| `onxza agent validate <agent-id>` | Run TORI-QMD validation on all 6 files in an agent workspace |
| `onxza agent retire <agent-id>` | Retire an agent — archive learnings, remove from registry, preserve workspace for audit |
| `onxza agent convert <agent-id>` | Convert a temporary agent to a persistent daemon |
| `onxza agent training [agent-id]` | Show training mode status — first 10 tasks tracked, quality metrics per agent |
| `onxza agent checklist <agent-id>` | Print pre-retirement checklist for an agent (dry-run, no changes made) |

**Examples:**
```bash
onxza agent create DTP_ONXZA_Architect
onxza agent list --company DTP
onxza agent validate dtp-onxza-architect
onxza agent retire dtp-onxza-cli
onxza agent training dtp-onxza-cli
```

## `onxza tickets`

List and manage ONXZA tickets.

```bash
onxza tickets [--status <status>] [--all] [--company <name>] [--assigned-to <agent-id>] [--priority <level>] [--project <slug>] [--type <type>]
```

| Subcommand | Description |
|---|---|
| `onxza tickets show <ticket-id>` | Display full ticket detail |
| `onxza tickets create` | Scaffold a new ticket with correct frontmatter and naming convention |
| `onxza tickets move <ticket-id> <status>` | Move a ticket to a new status directory (updates frontmatter) |

**Filters:**
```bash
onxza tickets --status open
onxza tickets --company DTP --priority high
onxza tickets --assigned-to dtp-onxza-cli
onxza tickets --all
```

**Create options:**
```bash
onxza tickets create \
  --summary "Build CLI release pipeline" \
  --type task \
  --assigned-to dtp-onxza-cli \
  --priority high \
  --company DTP \
  --project onxza
```

## `onxza learnings`

Shared Learnings promotion pipeline — push, review, and promote knowledge across tiers.

| Command | Description |
|---|---|
| `onxza learnings new <slug>` | Scaffold a new learning file from template |
| `onxza learnings push <file>` | Submit a learning file to a company-tier shared-learnings directory |
| `onxza learnings review` | Show pending `skill_approval_request` tickets (PM review queue) |
| `onxza learnings promote <file>` | Promote a company-tier learning to global (or cross-company) |
| `onxza learnings list` | List all shared learning files with metadata |
| `onxza learnings validate <file>` | Run TORI-QMD validation on a learning file before pushing |

**Examples:**
```bash
onxza learnings new my-pattern --company DTP --type pattern
onxza learnings push ./my-pattern.md --company DTP --type pattern --agent dtp-onxza-cli
onxza learnings review --company DTP
onxza learnings promote shared-learnings/DTP/patterns/my-pattern.md --to global --agent dtp-pm
onxza learnings list --company DTP
onxza learnings list --tier global --type skill
onxza learnings validate ./my-learning.md
```

**Learning types:** `pattern` | `correction` | `tool_note` | `escalation_log` | `model_observation` | `workflow` | `skill`

## `onxza checkpoint`

Manage ONXZA safety checkpoints — create snapshots before irreversible actions.

| Command | Description |
|---|---|
| `onxza checkpoint create <slug>` | Create a checkpoint snapshot before an irreversible action |
| `onxza checkpoint list` | List all checkpoints with timestamp and event slug |
| `onxza checkpoint verify <checkpoint-id>` | Verify vision document hashes against a stored checkpoint |
| `onxza checkpoint restore <checkpoint-id>` | Print step-by-step restore instructions for a checkpoint |
| `onxza checkpoint diff <id1> <id2>` | Show agent additions/removals between two checkpoints |

**Examples:**
```bash
onxza checkpoint create before-delete-agents
onxza checkpoint list
onxza checkpoint verify 20260318-191213-dtp-021-checkpoint-system-build
onxza checkpoint restore 20260318-191213-dtp-021-checkpoint-system-build
onxza checkpoint diff 20260317-180323-onxza-foundation-complete 20260318-144057-onxza-full-build-2026-03-18
```

## `onxza audit`

Manage the ONXZA append-only audit trail.

| Command | Description |
|---|---|
| `onxza audit log` | Append an irreversible action entry to the audit trail |
| `onxza audit list` | List audit trail entries with optional filters |
| `onxza audit verify` | Verify SHA256 checksums on all audit trail entries |

**Examples:**
```bash
onxza audit log --agent wdc-coo --action "Deleted 200 stale pages" --outcome ok --reversible no
onxza audit list
onxza audit list --agent wdc-coo --date 2026-03-18
onxza audit verify
```
