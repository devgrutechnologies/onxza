# Getting Started with ONXZA

> ONXZA is an AI company operating system. You define agents with roles, responsibilities, and governance rules — ONXZA runs them.

## Prerequisites

- **Node.js 18+**
- **npm** (or yarn)
- A terminal
- An API key for Anthropic or OpenAI *(optional if using a local LLM)*

---

## Step 1 — Install

```bash
npm install -g onxza
onxza --version
```

Expected output:
```
0.1.0
```

---

## Step 2 — Initialize a Workspace

```bash
onxza init my-company
```

This creates the ONXZA workspace at `~/.openclaw/`:

```
~/.openclaw/
├── openclaw.json         # Installation config + agent registry
├── workspace/
│   ├── tickets/          # Ticket system (open/, pending/, closed/)
│   ├── shared-learnings/ # Cross-agent knowledge base
│   └── docs/             # Governance documents
└── scripts/              # Automation pipeline
```

---

## Step 3 — Create Your First Agent

ONXZA agents follow a strict naming convention: `Company_Department_Role`.

```bash
onxza agent create MyCompany_Research_Analyst
```

Six files define every agent:

| File | Purpose |
|------|---------|
| `AGENTS.md` | Operating instructions — identity, responsibilities, ticket types |
| `SOUL.md` | Persona, voice, and decision-making style |
| `IDENTITY.md` | Machine-readable metadata (ID, company, model) |
| `MEMORY.md` | Persistent session memory — written at end, read at start |
| `TOOLS.md` | Available tools and usage guidance |
| `HEARTBEAT.md` | Cron-compatible execution spec |

---

## Step 4 — Assign Work

```bash
onxza tickets create \
  --summary "Research top 5 competitors" \
  --type task \
  --assigned-to mycompany-research-analyst \
  --priority high
```

Then run the dispatcher to notify the agent:

```bash
onxza dispatcher run
```

---

## Step 5 — Check Status

```bash
onxza status          # System overview
onxza agent list      # All registered agents
```
