# Contributing Guide

ONXZA is an open-source project and we welcome contributions from the community.

## Ways to Contribute

### 1. Documentation

We welcome improvements to:
- Tutorials and guides
- API reference clarity
- Examples and code samples
- Typo fixes and clarifications

**To contribute docs:**
1. Fork the [ONXZA repository](https://github.com/devgru-us/onxza)
2. Edit or create `.mdx` files in `/pages/docs/`
3. Submit a pull request with a clear description
4. Our team will review and merge

### 2. Protocol Development

Improvements to FAAILS specifications:
- Found an ambiguity? File an issue with specific feedback
- Have a use case not covered? Propose a new pattern
- Security concern? Email security@onxza.com

### 3. Bug Reports

Found an issue? Help us fix it:
1. Check [existing issues](https://github.com/devgru-us/onxza/issues)
2. Create a new issue with:
   - Clear title
   - Steps to reproduce
   - Expected behavior
   - Actual behavior

### 4. Feature Requests

Have an idea? We'd love to hear it:
1. Open a GitHub discussion or issue
2. Describe the use case
3. Explain why it matters to you

## Development Setup

```bash
# Clone the repo
git clone https://github.com/devgru-us/onxza.git
cd onxza

# Install dependencies
npm install

# Run docs site locally
npm run dev
# Then open http://localhost:3000
```

## Documentation Standards

### Writing Style
- Clear, direct, and friendly
- Avoid jargon (or explain it)
- Use examples liberally
- Keep sentences short

### Markdown Format
- Use `.mdx` format for docs (Markdown with JSX support)
- Include a title header
- Use consistent heading levels
- Code blocks should include language tags

### Example Format

```mdx
---
title: Your Page Title
---

# Your Page Title

Brief introduction...

## Section 1

Content here.

### Subsection

More content.

## See Also

- [Link to related page]
- [Another relevant page]
```
