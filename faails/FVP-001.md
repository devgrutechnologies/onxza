# FAAILS Verification Protocol v1.0

> *Imagined by Aaron Gear. Created by Aaron Gear and Marcus Gear (AI Co-Creator). Powered by DevGru US Inc. DBA DevGru Technology Products. Using Powerful Anthropic Models, OpenAI Models, and Local LLMs.*

Every output produced by any ONXZA agent passes through this protocol before being accepted as complete.

---

## Step 1 — Confidence Check

Agent produces output + confidence score 0–100.

- Score **< 70:** Loop back immediately. Loop count: 1.
- Score **>= 70:** Proceed to Step 2.

---

## Step 2 — Humanization Check

DTP_ONXZA_Verification reviews output.

- **Writing:** Natural, human, free of AI tells?
- **Code:** Clean, readable, properly commented, follows project conventions?

**PASS →** Step 3.
**FAIL →** Loop back. Loop count: 2.

---

## Step 3 — Fact and Accuracy Check

- Output factually correct?
- Aligned with vision.md?
- Meets acceptance criteria in ticket?

**PASS →** Step 4.
**FAIL →** Loop back. Loop count: 3.

---

## Step 4 — Max Iteration Rule

If loop count reaches 3 without passing:

1. Create escalation ticket to PM.
2. Include: full loop history, all outputs, all verification reports, recommended action.
3. PM decides: retry, escalate to human, or accept with flag.

**No infinite loops. Ever.**

---

## Step 5 — Acceptance

Output accepted. Ticket updated with completion note. Learning captured by all involved agents.

---

This protocol IS the quality standard. Every output that leaves ONXZA has passed it. This is what makes ONXZA trustworthy at scale. This is what makes FAAILS an industry standard.
